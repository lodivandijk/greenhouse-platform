# Domain Model

The domain model should describe the greenhouse in the language of growing and operating a controlled environment.

## Initial entities

### Greenhouse

Represents one controlled growing structure.

Possible attributes:

- Identifier.
- Name.
- Location.
- Dimensions.
- Orientation.
- Construction type.
- Operational status.

### Growing area

A physical area in which crops are grown.

Types may include:

- Bed.
- Trough.
- Pot.
- Shelf.
- Hydroponic channel.
- Commercial growing zone.

### Crop

Represents a crop type or variety.

Examples:

- Tomato.
- Strawberry.
- Cucumber.
- Basil.

### Planting

Represents a crop growing in a specific area during a specific period.

Attributes may include:

- Crop variety.
- Growing area.
- Planting date.
- Quantity.
- Growth stage.
- Expected harvest window.
- Actual harvest.

### Device

Represents a connected edge device.

Attributes:

- Device ID.
- Name.
- Role.
- Location.
- Firmware version.
- Last-seen time.
- Capabilities.
- Operational status.

### Sensor

Represents a measurement capability attached to a device.

Examples:

- Air temperature.
- Relative humidity.
- Soil moisture.
- Light.
- Water level.

### Actuator

Represents a controllable physical capability.

Examples:

- Pump.
- Valve.
- Fan.
- Heater.
- Vent.

### Observation

Represents an immutable measurement or reported event.

### Irrigation zone

Represents an independently controllable watering area.

### Automation policy

Represents the rules or strategy used to decide when action should occur.

### Recommendation

Represents advice generated by rules, analytics or AI.
